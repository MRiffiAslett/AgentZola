### Characteristics of TensorFlow Model for `ReshapeReshapeForwarding` Optimization

The `ReshapeReshapeForwarding` optimization pass triggers when specific conditions in the TensorFlow model are met. The model must have a sequence where a tensor undergoes two consecutive reshape operations such that the final shape of the tensor after these operations is identical to its original shape.

A TensorFlow model that would trigger this optimization typically involves:

1. An initial tensor with any arbitrary shape.
2. A first reshape operation that transforms this tensor to a new shape.
3. A second reshape operation applied on the result of the first reshape, which reverts it back to the original shape of the tensor.

```python

# Example input tensor of shape (4, 4)

# First reshape operation changes the shape

# Second reshape operation reverts to the original shape
reverted_tensor = tf.reshape(reshaped_tensor, [4, 4])
```

In this example, the `input_tensor` first gets reshaped to `[16]` and then is reshaped back to `[4, 4]`. This pattern is a direct match for the `ReshapeReshapeForwarding` pass, which will simplify this by removing the redundant reshapes.

### Characteristics of TensorFlow Model for `TupleSimplifier` Optimization

The `TupleSimplifier` optimization pass is triggered under specific structural conditions in a TensorFlow model that uses tuple and GetTupleElement (GTE) operations in a way that the entire tuple assembly and disassembly can be simplified.

Key characteristics for a model to trigger this optimization include:

1. A tuple is constructed using a series of operations.
2. Each element of the tuple is extracted using GTE operations.
3. The GTEs strictly extract elements in a manner where each element corresponds to the original operations that were used to form the tuple.
